# Token 使用效率技能（預設啟用）

## 核心原則

模型無法控制 input context，但可以控制**自己的輸出**與**工具呼叫行為**。

## Output 精簡

- 直接回答，不加前言後語
- 一句話說完不寫三段，不重複已知資訊
- 不替程式碼加註解，不總結做過什麼（除非被問）
- 不輸出感謝/禮貌用語
- 錯誤只摘要關鍵行號與類型，完整 trace 只在要求時給

## 工具呼叫效率

- 用 `grep` 取代多次 `read`，用 `glob` 取代 `ls`
- 不要為了找關鍵字而讀整份檔案
- 同一批獨立操作平行呼叫，不重複搜尋已知結果
- 一次讀取夠用範圍，避免連續 30 行小量讀取

## 檔案過濾

- grep 前用 glob 確認目標類型，避開 `.min.js`、`.bundle.js`、`.chunk.js`
- 不對 `.svg`、`.png`、`.ico`、`.woff2` 下 grep/read
- 不搜尋 `dist`、`build`、`.next`、`Pods`；`node_modules` 只在明確需要時
- 檔案 >500 行且只需特定函式：先用 grep 定位再用 read offset

## 上下文管理

- 已完成任務從 todo 移除
- 不主動重提舊版內容、不延續偏離主題的討論
- 長日誌輸出只摘要 1-3 行

## 使用方式

**全程啟用** - 每一輪回覆都應遵循上述原則。

強化模式關鍵字：「省 token」、「精簡回覆」、「減少輸出」、「不要廢話」
